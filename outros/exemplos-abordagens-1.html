Tabela 2: Categorias de Software, Descrição e Abordagem Típica para o Ciclo de Vida - Categorias 4 e 5.
Categoria Descrição Exemplos Típicos Abordagem Típica
Software Configurado (4) Software, geralmente mais complexos, que podem ser configurados pelo usuário para atender as suas necessidades especificas. O código do software não é alterado.
  • LIMS
  • Sistemas de aquisição de dados
  • SCADA
  • ERU
  • MRPII
  • DCS
  • CDS
  • EDMS
  • BMS
  • Planilhas
  • HMI (Human Machine Interfaces) simples
NOTA: Exemplos específicos de tipos de sistemas acima podem conter elementos customizados substanciais.
  • Abordagem de ciclo de vida
  • Abordagem para avaliação do fornecedor com base no risco
  • Demonstração de que o fornecedor possui um Sistema de Gerenciamento da Qualidade
  • Alguma documentação do ciclo de vida mantida pelo fornecedor (ex.: Especificações de Projeto)
  • Registro do número da versão e verificação da correta instalação
  • Realização de testes com base no risco para demonstrar que a aplicação funciona conforme projetada, em um ambiente de teste
  • Realização de testes com base no risco para demonstrar que a aplicação funciona conforme projetada, em um ambiente de produção
  • Existência de procedimentos para a manutenção do atendimento e adequação ao uso e gerenciamento dos dados
Software customizados (5) Software customizado projetado e codificado para atender a um processo de negócio. Inclui, dentre outros:
  • Aplicações de TI desenvolvidas internamente e externamente
  • Aplicações para controle de processo desenvolvidas internamente e externamente
  • Lógica de escada personalizada
  • Firmware personalizado
  • Planilhas (macro)
 Alem da abordagem utilizada para a Categoria 4, considera-se ainda:
  • Uma avaliação de fornecedor mais rigorosa, com possível auditoria deste
  • Posse da documentação de todo o ciclo de vida do sistema (Especificações Funcionais, Especificações de Projeto, testes estruturais etc.)
  • Revisão do Projeto e do Código Fonte
